# Database Schema

Neon PostgreSQL via Drizzle ORM. Schema lives in [`src/db/schema.ts`](../src/db/schema.ts) (app tables) and [`src/db/auth-schema.ts`](../src/db/auth-schema.ts) (Better Auth tables). Generate and apply migrations with `pnpm db:generate` / `pnpm db:migrate`.

**Conventions**

- **IDs** are cuid2 strings (`text`), generated app-side via `@paralleldrive/cuid2` — except `board_members` / `card_labels` (composite keys) and the auth tables (Better Auth supplies the id).
- **Ordering** uses `sort_order` — a **fractional-indexing string** (e.g. `"a0"`, `"a0V"`), not an integer. A drag writes only the moved row; siblings are never renumbered.
- **Soft delete** — cards archive via `is_archived`; they're never hard-deleted on archive.
- **App-layer references** — `board_members.user_id`, `cards.assignee_id`, `comments.user_id`, and `ai_rate_limits.user_id` point at `user.id` but have **no DB foreign key** (per SPEC §4); membership is always enforced in code via `board_members`.

## Application tables

### `boards`
| Column | Type | Notes |
| --- | --- | --- |
| `id` | text | PK, cuid2 |
| `name` | text | not null |
| `bg_color` | text | not null, default `#6366f1` |
| `created_at` | timestamptz | not null, default now |

### `board_members`
One row per (board, user). Role drives owner-only permissions.

| Column | Type | Notes |
| --- | --- | --- |
| `board_id` | text | not null, FK → `boards.id` **cascade** |
| `user_id` | text | not null, app-layer ref → `user.id` |
| `role` | text | not null, default `member`, enum `owner` \| `member` |

- **Unique** `uq_board_member` on (`board_id`, `user_id`).

### `columns`
| Column | Type | Notes |
| --- | --- | --- |
| `id` | text | PK, cuid2 |
| `board_id` | text | not null, FK → `boards.id` **cascade** |
| `name` | text | not null |
| `sort_order` | text | not null, fractional index |

- **Index** `idx_column_board` on `board_id`.

### `labels`
Board-scoped; shared across all cards on the board.

| Column | Type | Notes |
| --- | --- | --- |
| `id` | text | PK, cuid2 |
| `board_id` | text | not null, FK → `boards.id` **cascade** |
| `name` | text | not null |
| `color` | text | not null, hex |

### `cards`
| Column | Type | Notes |
| --- | --- | --- |
| `id` | text | PK, cuid2 |
| `column_id` | text | not null, FK → `columns.id` **cascade** |
| `title` | text | not null |
| `description` | text | nullable (Markdown) |
| `due_date` | timestamptz | nullable |
| `sort_order` | text | not null, fractional index |
| `is_archived` | boolean | not null, default false (soft delete) |
| `assignee_id` | text | nullable, app-layer ref → `user.id` |
| `created_at` | timestamptz | not null, default now |

- **Index** `idx_card_column` on `column_id`.

### `card_labels`
Join table — which labels are applied to which cards.

| Column | Type | Notes |
| --- | --- | --- |
| `card_id` | text | not null, FK → `cards.id` **cascade** |
| `label_id` | text | not null, FK → `labels.id` **cascade** |

- **Unique** `uq_card_label` on (`card_id`, `label_id`) — no duplicate label on a card.

### `comments`
| Column | Type | Notes |
| --- | --- | --- |
| `id` | text | PK, cuid2 |
| `card_id` | text | not null, FK → `cards.id` **cascade** |
| `user_id` | text | not null, app-layer ref → `user.id` |
| `content` | text | not null |
| `created_at` | timestamptz | not null, default now |

- **Index** `idx_comment_card` on `card_id`.

### `ai_rate_limits`
Per-user fixed-window limiter for `POST /api/ai/card-generate`. Postgres-backed so it stays correct across serverless instances with no external infra; the window resets lazily on the first request after it expires.

| Column | Type | Notes |
| --- | --- | --- |
| `user_id` | text | PK, app-layer ref → `user.id` |
| `count` | integer | not null, default 0 |
| `window_start` | bigint | not null, epoch ms |

## Better Auth tables

Generated by `@better-auth/cli` and managed by Better Auth — `user`, `session`, `account`, `verification`, plus `rateLimit` (the database-backed auth rate-limit store). The app's `user_id` / `assignee_id` columns reference `user.id` at the application layer.

| Table | Purpose | Key columns |
| --- | --- | --- |
| `user` | Accounts | `id` PK, `email` **unique**, `name`, `email_verified`, `image` |
| `session` | Login sessions | `id` PK, `token` **unique**, `user_id` FK → `user.id` **cascade**; index `session_userId_idx` |
| `account` | Credentials / providers | `id` PK, `user_id` FK → `user.id` **cascade**; index `account_userId_idx` |
| `verification` | Email/token verification | `id` PK, `identifier`; index `verification_identifier_idx` |
| `rateLimit` | Auth rate-limit store | `id` PK, `key`, `count`, `last_request` |

## Cascade chains

Deleting a row cleans up everything below it automatically:

```
boards ─┬─► board_members
        ├─► labels ─────────► card_labels
        └─► columns ─► cards ─┬─► card_labels
                              └─► comments

user ───┬─► session
        └─► account
```
